Unidades Responsivas

Son unidades de medida que se ajustan de forma dinámica en función del tamaño de la pantalla, la ventana del navegador o el contenedor padre.

Vmin y Vmax.
Estos toman el valor mínimo y máximo entre el ancho y la altura de la ventana, respectivamente, lo que permite una adaptabilidad mayor en ocasiones donde la orientación de la pantalla puede variar.

Unidades Dinamicas de Vista de Ventana

Las unidades dinámicas ofrecen mayor precisión en el diseño responsivo porque toman en cuenta las variaciones de tamaño de la ventana grafica en dispositivos móviles. Al ser unidades relativamente nuevas el soporte del navegador puede variar. Las unidades dinámicas de vista de ventana son:
    dvh
    svh
    lvh
    dvh
Dynamic viewport height (altura del viewpor dinamico), representa el 1% de la altura de la ventana grafica dianmica, ajustándose en tiempo real a los cambios de la interfaz, como la aparición o desaparición de barras de navegación.
Small viewport heigth (altura del viewport pequeño) y corresponde al 1% de la altura de ventana gráfica más pequeña que el navegador a registrado. Large Viewport Heigth, se refiere al 1% de la altura de la ventana gráfica registrada, permitiendo diseñar para el máximo espacio disponible.

Mobile First

El enfoque mobile first en diseño web adaptativo se centra en desarrollar inicialmente la interfaz para dispositivos móviles pequeños, como teléfonos inteligentes, y luego ampliarla progresivamente para pantallas más grandes, como tablets y escritorios.
Este método asegura que el contenido esencial y la funcionalidad se mantengan optimizados para usuarios móviles, que suelen ser la mayoría, antes de agregar elementos más complejos que requieren pantallas más amplias. Ventajas del enfoque Mobile First
Optimización de contenido y velocidad. Al centrarse en lo esencial desde el inicio, se logra que la página cargue más rápido en dispositivos con menos capacidad de procesamiento y conexiones de red más lentas.
Mejora de experiencia de usario en moviles. Al priorizar lo móvil, el diseño se adapta mejor a las necesidades de los usuarios que utilizan sus teléfonos.

Notas

Al centrarse en lo esencial desde el inicio, se logra que la página cargue más rápido en dispositivos con menos capacidad de procesamiento y conexiones de red más lentas.

Modulo Grid

El objetivo es conocer las configuraciones para el contenedor principal del módulo grid y también las configuraciones de sus hijos. En el pasado se usaban las propiedades table, luego float, position y display, sin embargo, estos métodos omiten funciones importantes como el centrado vertical. Grid es un sistema de diseño bidimensional basado en cuadriculas que en comparación con cualquiera de los sistemas de diseño web. Fue creado para resolver problemas de diseño, a diferencia de flexbox cuyo flujo unidireccional tiene diferentes casos de usos. Grid permite definir un esquema de cuadriculas. Display: grid; Para comenzar se debe definir un elemento con propiedad display: grid; Grid Este trabaja a travez de columnas de froma de vertical y renglones de manera horizontal.
Grid Tamplate areas
Propiedad de CSS utilizada en CSS Grid Layout para crear y organizar diseños de cuadrícula (grid) de una manera visualmente intuitiva. Permite definir áreas nombradas dentro de la cuadrícula, asignando a cada área un nombre único que luego puede ser referenciado para posicionar elementos.

Notas

Con la propiedad Grid-template-areas es posible crear una maqueta completa escribiendo como si fuera una matriz. Una vez declarada el acomodo de la página es necesario asignarle el nombre que corresponde.

@Supports

La regla @supports en CSS permite detectar si el navegador soporta una caracteristica de CSS especifica y aplicar estilos adicionales basados en esta compatibilidad. Esto es util pra implementar funcionalidades modernas mientras se garantiza que los navegadores que no soportan estas caracteristicas aún presenten u diseño funcional. La estructura basica de @supports es la siguiente: @supports (propiedad: valor){
elemento{
propiedad: valor;
}
}
La regla @supports puede utilizar los siguientes operadores logicos:

    and: para que la condición se cumpla ambas proposiciones deben ser verdaderas
    or: para que la condición se cumpla al menos una de las proposiciones debe ser verdadera
    not:: para que la condición se cumpla, la proposición debe ser falsa

Mejora progresiva y degradación elégante


Mejora Progresiva
La mejora progresiva es una estrategia de diseño que consiste en crear una versión básica de la funcionalidad para todos los navegadores, para luego agregar caracteristicas mejoradas.
Degradación elegante
Se refiere a la estrategia de crear la versión mas avanzada de una funcionalidad y asegurarse de que se degrade de manera aceptable.

Notas

No tenemos manera de saber desde que navegador se visita nuestra web, es por eso que es ideal darle soprte en navegadores mas antiguos que no aceptan propiedades mas nuevas, la regla supports permite darle estilos adicionales.
